## FIX 1 — ยอดติดลบ (หักซ้ำรายการวันนี้)  ⭐ หลัก
**Root cause:** หน้าบันทึกยอด (`page_daily_balance.jsx:230`,`:282`) ตอน save ยอด "พรุ่งนี้" → เซ็ต `bankAccounts.BALANCE = ยอดที่กรอก`, `DATE = พรุ่งนี้`
แล้วการ์ด `bdBuildAccountView()` (`page_bank_diary.jsx:178-179`):
```js
const asOfRef = (acct.asOf && acct.asOf < today) ? acct.asOf : today;  // ← บั๊ก: cap ที่ today
const base    = acct.balance;                                          // = ยอดพรุ่งนี้ ✅ ถูกแล้ว
```
พอ `asOf = พรุ่งนี้` (อนาคต) → `asOf < today` เป็นเท็จ → ตกมาใช้ `today` → ยังลากรายการ "วันนี้" (จ่ายไปแล้ว) มาหักซ้ำ → ติดลบ

**แก้ (บรรทัดเดียว):**
```js
const asOfRef = acct.asOf || today;   // ใช้วันที่ของยอดที่บันทึก ไม่ cap ที่ today
```
**ขับด้วย "วันที่ของยอดที่บันทึก" ไม่ใช่นาฬิกา** (ผู้ใช้ยืนยัน — ไม่ได้เข้ามาทุกวัน):
- 🌅 บันทึกยอด "วันนี้" → เห็นวันนี้+อนาคต (เหมือนเดิม)
- 🌆 บันทึกยอด "พรุ่งนี้" → รายการวันนี้หลุดเอง เริ่มจากพรุ่งนี้บนยอดใช้ได้จริง → ไม่ติดลบ
- ไม่ได้อัปเดต / ข้ามหลายวัน → asOf ค้างที่วันล่าสุด เห็นทุกอย่างตั้งแต่วันนั้น (ไม่มีอะไรหลุดเองจากเวลาผ่านไป)
- ✅ **ไม่ต้องมี toggle**

## FIX 2 — เช็คค้างขึ้นเงิน ต้องไม่หาย
ผู้ใช้ยืนยัน: เช็คปกติขึ้นตรงวัน (จ่ายไม่กี่บ.) แต่ **บางบ.เอาไปขึ้นทีหลัง/ติดวันหยุด**
- ปัจจุบัน filter `:184` = `bdIsOutstanding(c._st) && c.checkDate >= asOfRef` → เช็ค outstanding ที่เลยวันมาแล้วจะหลุด → ยอดใช้ได้ดูเยอะเกินจริง
- **แก้:** เช็ค outstanding (ยังไม่ขึ้นเงิน) **โชว์เสมอไม่ว่าวันไหน** → เอาเงื่อนไข `checkDate >= asOfRef` ออก (เหลือแค่ `bdIsOutstanding`)
- เช็คที่ลงวันที่ < asOfRef → clamp ให้แสดง/หักที่ต้นช่วง เป็นกลุ่ม "เช็คค้างขึ้นเงิน (ยกมา)" กันยอด running เพี้ยน
- (เช็ค cleared ยังตัดออกเหมือนเดิม — เงินออกจริงแล้ว อยู่ใน balance)

## FIX 4 — รายการจ่ายในแต่ละวัน: ย่อให้สวย + โชว์ REMARK
จุดเดียวกัน: `BDDayGroup` render (`page_bank_diary.jsx:744-762`) + `BDItemRow` (`:651`) + `BDDayItemGroup` (`:676`)

**(ก) ย่อรายการเดี่ยวให้เหมือนกลุ่ม (สวยงาม/เป็นแถวเดียว)**
- ปัจจุบัน `:756-760`: group ที่มี 1 ใบ → `BDItemRow` (แถวแบน โชว์ AP# + ดินสอ inline 2 บรรทัด), >1 ใบ → `BDDayItemGroup` (▶ ย่อ + "N ใบ" สะอาด)
- ผู้ใช้อยากให้รายการ 1 ใบ **ย่อแบบเดียวกับกลุ่ม** → ทุกแถวเป็น header สะอาด (▶ + tag + ชื่อ + ยอดรวม) กดกางดูรายละเอียด
- แก้: route ทุก group (รวม `length === 1`) ผ่าน `BDDayItemGroup`; เมื่อ count=1 จะโชว์ "1 ใบ" หรือซ่อน count ก็ได้ (เลือกตอนทำ); detail = `BDItemRow` เดิม (AP# + remark + ✏️)
- nuance: check/PV/transfer (ไม่มี `group`) จะกลายเป็น "1 ใบ" ด้วยเพื่อ uniform — ถ้าดูแปลกค่อยยกเว้นเฉพาะ forecast ตอน impl

**(ข) AP (ประมาณการจ่าย) โชว์ REMARK**
- ปัจจุบัน forecast item (`bdBuildAccountView:199-203`) โชว์ `title=desc`, `sub=สถานะ + AP#` — **ไม่มี remark**
- remark ของ AP อยู่ที่ `payables.remark` (`bdNormAP:171`); forecast ผูก AP ด้วย `refDoc = REF_DOC = vchno` (`:202`, `:1610`/`:1637`) — ตอนวางแผน `NOTE` ถูกตั้ง `null` (`:1636`) → **ต้องดึง remark จาก payables เอง**
- แก้: สร้าง map `vchno → remark` จาก `payables` → enrich forecast item: `it.remark = payables[refDoc]?.remark || forecast.NOTE || ''`
- แสดง: ใน **รายละเอียดที่กางออก** ของแต่ละใบ — เพิ่มบรรทัด remark ใต้ AP# (`BDItemRow` เพิ่ม line `it.remark`)
- collapsed header ยังสะอาด (vendor + ยอด) — remark โผล่ตอนกาง
- 🔸 sub-decision: ถ้าผู้ใช้อยากเห็น remark **โดยไม่ต้องกด** → โชว์ remark เป็นบรรทัดเล็กใต้ชื่อในหัวที่ย่อด้วย (ค่อย confirm)
