## No build step — in-browser Babel (read this first)
There is **no bundler, no package.json, no npm, no tests, no lint**. `index.html` loads React 18 + `@babel/standalone` from CDN, then loads every source file as `<script type="text/babel" src="app/xxx.jsx?v=DATE">`. Babel transpiles JSX **in the browser** at page load.

Consequences:
- Every top-level `function`/`const` in a `.jsx` is a **global** shared across all files — there are **no imports/exports**. (`fmtNum`, `Modal`, `WTPData`, `WTPOverride`, `categorizePayable`, etc. are all global.)
- **Adding a new `.jsx` → add a `<script type="text/babel">` tag in `index.html`.**
- **Editing any `.jsx`/`.js`/`.css` → bump its `?v=` query in `index.html`** (e.g. `?v=20260609a` → `...b`). This is the cache-bust; without it, returning browsers serve the cached old file even after deploy. **Every edit pairs with a `?v=` bump.**

## Run / preview locally
Must be served over **HTTP** (not `file://`) — it fetches ~25 separate `.jsx`.
- `python -m http.server 8000` from repo root, open `http://localhost:8000` (entry = **`index.html`**). No Node on this machine.
- `.claude/launch.json` has an `http-server` config (port 8000) for preview tooling.
- The app has a **login gate**: a valid `wtp-session` in `localStorage` is required; data loads from the synced Sheet (cached in `localStorage` key `wtp-fin-data-v8`). To preview without real login, inject a session via the page console.

  - **All dates display as `DD/MM/YYYY` CE** (`PCU.fmtDate` short). `PCU.isoOf` robustly parses the engineer file's **mixed** date formats → ISO: `Date` object, `22/May/26`, ISO, and `n/n/n` with 2- or 4-digit year. D/M vs M/D disambiguation: a part >12 is the day; if ambiguous, **4-digit year ⇒ D/M/Y** (Thai-typed text), **2-digit year ⇒ M/D/Y** (Excel US auto-format, e.g. `8/30/24`→30/08/2024, `2/11/26`→11/02/2026). `parseProjectControl` normalises **all** `pcColType==='date'` columns via `isoOf` on import (`DATE_COLS` derived from `PC_COL_SPEC`), and grid raw date columns return ISO from `value` so sort/filter/display are all consistent.
    - `PCU.openReport(kind, rows, scopeLabel, cols)` kind `'summary'|'detail'|'both'` — styled HTML window (brand-blue, **Water POG logo** via `new URL('waterpog_Logo-02.png', location.href)`, print-color-exact); `'both'` concatenates summary + `page-break-before` + detail. `buildDetailBody()` renders the `cols` table (money right/฿, pct·date center, ค้างรับ amber). Summary = KPI tiles + pipeline + monthly cashflow per year + LG.

PC-prefixed helpers `pcXlSet/pcXlRow/pcXlFmt` + `PCXL` style map.

Fixed projects (e.g. ENC165) that had only POG + dates but no งวด columns showing a stuck 20%.

## Data model & overrides
- Entity tables: `bankAccounts`, `payables` (AP), `pvVouchers` (PV), `forecastEntries`, `invoices` (IV), `receipts`, `checks`, `debtMaster`, `debtLedger`, `bankEntries`, `bankTransfers`, `cashflowSnapshots`, `projects`, `users`.
- **`WTPOverride`** — a synced key→value KV store (`data.manualOverrides`) for UI tweaks that must NOT mutate backend entity tables: cashflow edits (`cf.*`), per-PV category (`cf.pvCat.<PL_PV_No>`), IV plan-lock (`*.ivPlan.*`), Project Control Finance Master (`pcfin.<ContractNo>` = JSON string). API: `resolve(key, fallback)`, `set` (coerces to Number), **`setRaw`** (keeps string/JSON — used by `pcfin.*`), `setMany`, `clear`, `has`, `_load()` (returns merged cloud+local map). (`cf.paidAp.*` manual-paid flags are **deprecated** — removed from cashflow when AP planning moved to Bank Diary.)
- **Roles**: `viewer`/`owner` = read-only (`_wtpRoleIsReadOnly()` true, `WTPAuth.can('canEdit')` false); `manager`/`staff` can edit. Gate all edit UI behind `canEdit`.

## Conventions & gotchas
- **Cache-bust `?v=`** on every edit (see above) — easiest thing to forget, breaks deploys silently.
- Sync is **last-write-wins per entity** across users; many guards stop accidental mass-delete/clobber. Never push a stale-low or emptied table. Clearing a field over sync requires it in `CLEARABLE_FIELDS` / `mergeRowKeepSheetForEmpty` (else it bounces back).
- Some date fields are **DD/MM/YYYY (Thai)** — normalize to ISO before `new Date()` or months flip.
- Cashflow categories (หมวด 1-4): `categorizePayable(ap)` / `categorizeForecastEntry(fe)`, plus vendor→category map `cfVendorCat` (localStorage `wtp-cf-vendor-cat`, e.g. ลีซ อิท → 3) and per-PV override `cf.pvCat.<PL_PV_No>`.
- **Cashflow = Forecast vs Actual, ไม่ยุ่งกับ AP (since 2026-06-11).** Outflow **Forecast** (`forecastByWeekCat`) = **`forecastEntries` only** (manual, ตั้งจากหน้า Forecast: non-LOAN, outflow `AMOUNT<0`, STATUS≠CANCELED) — AP-by-due-date is **no longer** pulled into cashflow (fixes the AP+forecast double-count). Also the cashflow `forecastEntries` var is **filtered to drop `EXPENSE_TYPE='AP'`** — those are forecast rows Bank Diary's `BDApPanel` writes when you plan an AP payment; they'd otherwise double-count against the manual lump estimates. **Actual** (`pvActualByWeekCat`) = `pvVouchers` by `Pmt_Date` (category via `resolvePvCategory`, NOT default-cat-1) + `forecastEntries` STATUS∈{ACTUAL,BOOKED}. The old flex-pool ("จ่ายตามสภาพคล่อง") + manual-paid (`cf.paidAp`) machinery was **removed** from cashflow; AP planning (ครบดิว/เลือกจ่าย/เลื่อนจ่าย) lives in **Bank Diary** (`BDApPanel`). KPI / Plan-table / weekly-tracking all derive from the **same** `forecastByWeekCat` grid so top ↔ bottom numbers match. Both Forecast and Actual weekly cells are click-drillable (`openForecastDrill` / `openActualDrill`). **Weekly tracking card (section 02) per-week progress bar:** the `%` label **rides the bar's leading edge** (`left: min(100,pct)%`, white-inside when fill ≥40% else dark-outside-right) and caps at the right end when ≥100%; each card's footer shows a small soft pill **`▲ เกินแผน ฿X`** (red, actual>plan) / **`▼ เหลือ ฿X`** (green, actual<plan) tucked under "Total Paid :" — `diff = planTotal − actualTotal`.
- Cashflow IV forecast excludes already-received IVs (`ivIsPaid`); a day-1 "plan lock" freezes the monthly baseline so PLAN doesn't shrink as IVs get paid.
- **AP paid via PV ⇒ drop its forecast (Bank Diary only now).** An AP whose number matches a `pvVouchers.AP_No` is "paid". **Bank Diary** uses a global `paidApSet` (built from `pvVouchers.AP_No`) applied in **both** the account card (`bdBuildAccountView`) **and** the forecast panel (`BDForecastPanel`) — filters out any forecast (ประมาณการ) whose `REF_DOC ∈ paidApSet` so the real PV row represents the outflow (else the plan lingers / double-counts vs the already-reduced BALANCE). Both spots must stay in sync when touching forecast logic. (Cashflow no longer pulls AP, so its `buildPaidVchnoSet`/`isApPaid` were removed.)
- **AP amount = net (`netpayment`).** Bank Diary `bdNormAP` and the AP panel show `netpayment` (ยอดสุทธิ), falling back `Amount → net_new → Balance_Amount1` — matches Cashflow's `Number(ap.netpayment || ap.Amount)`. Don't show `Balance_Amount1` (ยอดคงเหลือ) as the AP amount.
- **Sortable+filterable tables**: reuse the shared `FilterableColHeader` + `ColFilterDropdown` (in `components.jsx`) with parent-owned `colFilters`/`openCol` state and a `getValue(row, colKey)` (see [[colfilter-getvalue-convention]] — must be `(row, key)`). Used by `page_invoices` and the Bank Diary AP panel. For date/number columns also pass the optional **`getSortValue(row, colKey)`** so the filter dropdown lists values in real order (return ISO `YYYY-MM-DD` for dates, the raw `number` for amounts) instead of string-sorting the formatted display (DD/MM/YYYY / `1,000` sort wrong). The AP panel uses `tableLayout:fixed` + fixed col widths (vendor flexes) with `…` ellipsis on long text cells. Bank Diary account-card footer shows **net ending balance** (`base + visIn − visOut` over the shown period), not lowest-balance; overdue-checks modal rows expand to a detail panel; ReconcilePanel is column-sortable.
- **Bank Diary card = period-scoped; everything respects the selected period (`periodEnd`), never a fixed 7-day window.** Shortage alert `isShort = shortInPeriod` (red border/badge/"ต้องเติมเงิน" strip + the `shortAccounts` KPI) — NOT `shortNear`; badge reads "ไม่พอในช่วง `periodLabel`". The "📆 ภายใน 7 วัน" chip only shows when `next7 <= periodEnd` (hidden for sub-7-day views like "สัปดาห์นี้"). When changing alerts, keep them period-based or they "leak" next-week items into a this-week view.
- **Save card as PNG** (`BankAccountCard` 📷 button): `html2canvas-pro` (already loaded in `index.html`, global `window.html2canvas`) on `cardRef`; `ignoreElements` skips `[data-no-capture="1"]` (the button itself); output is **cropped to the `[data-capture-end="1"]` footer bottom** + a small pad and accent finish line (red if `netEnding<0`, green if ≥0) so it doesn't look torn. Same html2canvas pattern as `page_daily`/`page_invoices`/`page_pnl`.
- **Transfer-item label is direction-aware**, computed from the leg's own direction — NOT `bankEntry.description` (the stored note can be the wrong-side "โอนเงินไป …" on both legs). `bdBuildAccountView` takes a `transferInfoByRef` (transferRef → {fromNo/fromBank, toNo/toBank}, built from `transferPairs` + `accounts`); inflow leg → "รับโอนจาก «from»", outflow → "โอนเงินไป «to»". A genuine user note (description not starting with "โอนเงินไป"/"รับโอนจาก") is kept as the item `remark`.
- **Bank Reconciliation page** (`app/page_bank_recon.jsx`, route `#bank_recon`, hidden from `owner`/`viewer` like other bank screens) — กระทบ PV ในระบบ กับ statement จริง ราย **บัญชี** ราย **เดือน**. Imports **CSV + Excel** (KTB) via the already-loaded global `XLSX`; a **column-mapping modal** (`BRMappingModal`) maps date / amount-or-(debit+credit) / desc / ref / balance per **bank brand** (auto-detect `brBrandKey`, saved per brand in `localStorage wtp-bankrecon-map-v1`) → works for any bank, no hardcoded parser. Statement lines + reconcile state are **localStorage v1** (`wtp-bankrecon-stmt-v1` / `-state-v1`, via `BankReconStore`), **not** synced — re-importable working data; the data-loss-prone Sheets sync stays untouched. `brReconcile` matches statement outflows ↔ PV by amount (±0.01) + date (±3d) + cheque/ref, with a subset-sum pass for 1-line-=-many-PV; **3 buckets**: matched / **missing (ขาดบันทึก)** / unmatched-PV. **Feedback loop:** "บันทึกจ่ายจริง" on a missing line writes a `forecastEntries` row tagged **`EXPENSE_TYPE='BANK_RECON'`** + `STATUS='ACTUAL'` (+ `ACTUAL_AMOUNT`/`ACTUAL_DATE`/`CATEGORY`/`Bank_AC`) → appears as **Actual on Cashflow** (and "จ่ายจริงแล้ว" in Bank Diary) with zero new plumbing. **⚠️ These rows are ACTUAL-only — they were never a plan, so cashflow's ประมาณการ (forecast) computations MUST exclude them or the forecast double-counts** (a recon-recorded payment would show in both Forecast and Actual). `page_cashflow.jsx` skips `EXPENSE_TYPE==='BANK_RECON'` in **both** `forecastByWeekCat` and the next-month rollover forecast loop (the `AMOUNT` field is kept negative so the sign stays right in Bank Diary's `bdNormForecast`); `pvActualByWeekCat` is left untouched so they still count as Actual. Fix is at the compute layer, so it retroactively de-double-counts the 15+ recon rows already in the sheet — no re-recording needed. **The drill-downs must mirror this:** `openForecastDrill` ALSO skips `BANK_RECON` (else the click-through item list exceeds the corrected cell total), and `openActualDrill` labels recon rows **`source:'STM'`** (badge reads "STM" + note "กระทบยอด (STM)") instead of "Forecast" — they're real payments, not a plan. The `PlanVsActualCard` lock badge (`🔒 ล็อกแผน …` + ล็อกใหม่ button) was moved to the **card footer** (below the progress bar, with a `borderTop` divider) instead of right under the header. (2026-06-14) Reuses `bd*` globals from `page_bank_diary.jsx` (`bdAcct`, `bdNormPV`, `bdAcctMatchesCheck`, `bdLast4`, `bdBrand`). `brMonthlyView` sorts by date + **orientation-aware** file index (handles newest-first files like BBL) so running-balance/opening/closing stay correct. **Real-file specifics (presets in `BR_PRESETS`, from actual บัญชี exports):** KBANK/BBL CSVs are **TIS-620/CP874** (not UTF-8) → `brDecodeText` (UTF-8 strict, fallback `windows-874`); Thai-month dates ("01-มิ.ย.-2569") via `BR_THAI_MONTH`; header row auto-found by `brGuessHeaderRow` (row with most header-like cells — `blankrows:false` drops blank rows so a fixed index shifts); **SCB** file holds *multiple accounts* → `acctCol` filters to the selected account, and its **Note column = the PV number** → `brReconcile` **pass 0** matches by PV-ref (strongest signal, handles 1-PV-many-legs). **KTB** exports an **encrypted .xls** (มีรหัสผ่าน) — `brParseFile(file, password)` ส่ง `password` ให้ `XLSX.read` (SheetJS 0.18.5 ถอด RC4/Standard ได้). เมื่อ `XLSX.read` โยน error `password/encrypt/protect` → เด้ง **`BRPasswordModal`** ให้ใส่รหัส → `parseAndPreview(file, pwd)` อีกครั้ง (รหัสผิด = วนถามใหม่ · ถอดไม่ได้/รูปแบบไม่รองรับ = โชว์คำแนะนำ Save As `.xlsx/.csv` ไม่ใส่รหัสเป็น fallback). รหัสใช้ในเครื่องเท่านั้น ไม่เก็บ/ไม่ขึ้น cloud.
